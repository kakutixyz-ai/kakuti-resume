# SmartResume — Modern Resume Editor
## 🌐 Live Site: [resume.kakuti.site](https://resume.kakuti.site/)
https://github.com/user-attachments/assets/8bbed2f0-304f-4ae3-a38d-eeec7b8676bc

SmartResume is a **fully browser-based, backend-less resume editor** that combines a **Notion-style block editor** with the **Typst typesetting engine** compiled to WebAssembly. Create, edit, and export professional resumes — all running locally in your browser with zero server dependencies.

**Three things that make it special:**
- 📄 **Auto-parse PDF resumes** — Upload an existing PDF and the editor automatically extracts content into editable blocks, no manual retyping needed.
- 🔒 **100% local, privacy first** — Everything runs in your browser. No uploads, no servers, no accounts. Your data never leaves your device.
- ✍️ **Markdown → PDF, free & effortless** — Write in familiar Markdown format, export straight to professional PDF. No formatting wrestling, no subscriptions.

## ✨ Key Features

### 🚀 Browser-side PDF Compilation (WASM)
All PDF generation runs directly in your browser via Typst compiled to WebAssembly inside a Web Worker. No backend, no queue, no cloud rendering. Your data never leaves your device.

### 📝 Notion-style Block Editor
A familiar block-based editing experience inspired by Notion. Supports heading levels (H1/H2/H3), bullet lists, and paragraph blocks with **rich text capabilities** including bold, text color, and font size customization. Keyboard shortcuts for seamless navigation and block manipulation.

### 📋 Five Professional Templates
- **Classic** — Minimal black-and-white, elegant and timeless
- **Modern** — Colored accents inspired by Awesome CV
- **Art** — Dark-themed artistic style with vibrant highlights
- **履歴書 (Rirekisho)** — Traditional Japanese resume in standard JIS format
- **職務経歴書 (Shokumukeirekisho)** — Japanese career history document

### 💾 100% Local & Private
Your resume data is saved in real-time to your browser's **IndexedDB** via localforage. No registration, no accounts, no cloud storage. Everything stays on your device.

### 📂 Import & Export
- **Import PDF** — Upload an existing PDF resume and let the built-in parser (powered by pdfjs-dist) extract content into editable blocks
- **Import Markdown** — Load from `.md` backup files with YAML frontmatter
- **Export Markdown** — Save your resume as a portable `.md` file
- **Export PDF** — Download a typeset PDF generated by Typst

### 📱 Responsive Design
Fully responsive layout with mobile-friendly edit/preview toggle. Works on desktop, tablet, and mobile devices.

### 🌏 Multi-language Support
UI is in English. Resume templates support **English** and **Japanese** content with appropriate fonts (Noto Sans CJK JP/SC) loaded automatically.

## 🏗 Architecture

### Fully Client-Side
The entire application is a **Single Page Application (SPA)** with no backend. All computation happens in the browser:

```
User Input → React State → Markdown/Typst Source → Web Worker → WASM Compilation → PDF/SVG → Preview
```

### Typst WASM Compilation Pipeline

```
┌─────────────┐     ┌──────────────┐     ┌─────────────┐     ┌──────────┐
│  Editor     │────▶│  Generator   │────▶│  Web Worker │────▶│  PDF     │
│  State      │     │  (Typst src) │     │  (typst.ts) │     │  Blob    │
└─────────────┘     └──────────────┘     └─────────────┘     └──────────┘
                                                 │
                                                 ▼
                                          ┌──────────┐
                                          │  SVG     │
                                          │  Preview │
                                          └──────────┘
```

1. **Editor State** — Personal info + content blocks managed by React state
2. **Generator** — Converts state into Typst source code (per template: western, rirekisho, shokumukeirekisho)
3. **Web Worker** — Initializes Typst WASM compiler and renderer, preloads fonts (Roboto, Noto Sans CJK, Font Awesome), manages template file loading
4. **Output** — Returns both PDF bytes (for download) and SVG pages (for live preview)

### PDF Import Pipeline

```
Upload PDF → pdfjs-dist → Text Extraction → Line Processing →
Header/Footer Detection → Section Analysis → Personal Info Extraction →
Body Parsing (H1/H2/H3/P) → Markdown Assembly → Editor Blocks
```

The parser supports template-specific parsing strategies for western resumes, rirekisho, and shokumukeirekisho formats.

### Local Persistence
- **Storage**: IndexedDB via `localforage`
- **Schema**: v2 with automatic migration from legacy keys
- **Auto-save**: Debounced at 800ms
- **Photos**: Stored separately as base64 data URLs

### State Management
No external state library. The `useEditorState` hook centralizes all state (personal info, content blocks, template, settings, photo) and coordinates sub-hooks for persistence, compilation, IO, photo management, and PDF import.
